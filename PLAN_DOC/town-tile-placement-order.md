# town.tscn 실제 타일 배치 작업 순서

## 문서 목적
- 현재 `town.tscn`의 단색 배경 플레이스홀더를 실제 마을 타일 배치로 바꿀 때의 순서를 정리한다.
- `Franuka` 또는 다른 타일셋을 쓰더라도 작업 순서를 흔들리지 않게 만든다.

작성일: 2026-04-15

---

## 1. 현재 전제
- `town.tscn`은 현재 `ColorRect` 배경 위에 기능 오브젝트만 배치된 상태다.
- 핵심 오브젝트:
  - `SavePoint`
  - `ShopNPC`
  - `SkillShopNPC`
  - `BlacksmithNPC`
  - `PortalToField1`
- 이 문서는 타일 배치를 어디서부터 해야 효율이 큰지 정리한 것이다.

---

## 2. 가장 먼저 할 일
- 단색 배경을 바로 장식으로 채우지 않는다.
- 먼저 아래 세 가지를 확정한다.
  1. 메인 길
  2. 중앙 광장
  3. 상점가 라인

이 세 가지가 먼저 잡혀야 나머지 소품이 의미를 가진다.

---

## 3. 추천 배치 순서

### Step 1. 바닥 전체 깔기
- 마을 기본 바닥 타일을 전체에 먼저 깐다.
- 이 단계에서는 장식 타일을 넣지 않는다.
- 화면 전체의 기본 톤만 만든다.

목표:
- `여긴 풀밭인가`, `돌바닥인가`, `흙바닥인가`를 먼저 정한다.

### Step 2. 메인 길 만들기
- `Player spawn`에서 `PortalToField1`까지 이어지는 메인 길 생성
- 길은 가능한 한 눈으로 바로 따라갈 수 있어야 한다.
- 너무 구불거리게 만들지 않는다.

목표:
- 플레이어가 처음 들어왔을 때 출구 방향을 직관적으로 알 수 있게 함

### Step 3. 중앙 광장 만들기
- 메인 길 중간 또는 상단 중앙부에 작은 광장 생성
- 광장은 상점가와 세이브 구역의 기준점 역할을 한다.

목표:
- 마을의 중심이 생기게 함

추천 요소:
- 돌바닥 확장
- 우물
- 큰 나무
- 조형물

### Step 4. 세이브 구역 바닥 구분
- `SavePoint` 주변만 다른 바닥 타일이나 경계 타일로 분리
- 상점가보다 더 조용하고 안정적인 느낌을 준다.

목표:
- 세이브 포인트가 허공에 떠 있는 오브젝트처럼 보이지 않게 함

### Step 5. 상점가 라인 만들기
- `ShopNPC`, `SkillShopNPC`, `BlacksmithNPC` 뒤에 배경 구조물 또는 가판대 배치
- 세 NPC가 같은 줄에 있어도 역할이 다르게 보이도록 소품을 분화

예시:
- `ShopNPC`: 상자, 천막, 배럴
- `SkillShopNPC`: 책, 배너, 룬 장식
- `BlacksmithNPC`: 모루, 화로, 무기 거치대

목표:
- 상점가가 단순 NPC 나열이 아니라 실제 기능 구역처럼 보이게 함

### Step 6. 출구 구역 꾸미기
- `PortalToField1` 주변에 관문/성문/다리 입구 같은 시각 요소 추가
- 포털이 아무 이유 없이 맵 끝에 서 있는 느낌을 없앤다.

목표:
- 바깥 필드로 나가는 통로처럼 보이게 함

### Step 7. 외곽 경계 만들기
- 맵 경계에 나무, 울타리, 절벽, 건물 벽면 등 배치
- 현재 충돌벽을 시각적으로 설명하는 역할

목표:
- 플레이어가 왜 더 못 가는지 납득하게 함

### Step 8. 소품 최소 추가
- 마지막에만 소품을 넣는다.
- 상자, 화분, 램프, 간판 정도로 시작
- 과하게 채우지 않는다.

목표:
- 공간이 살아 있으면서도 동선 가독성을 해치지 않게 함

---

## 4. 레이어 작업 우선순위
타일/오브젝트를 분리한다면 아래 순서를 권장한다.

1. Ground
2. Road / Plaza
3. Structure / Boundary
4. Props
5. NPC / Interactive

중요:
- 인터랙션 오브젝트보다 바닥과 구조를 먼저 완성한다.

---

## 5. 하지 말아야 할 순서
- 소품부터 예쁘게 놓기
- 포털 주변을 마지막까지 비워두기
- 세이브 구역과 상점가를 같은 바닥 톤으로만 처리하기
- 외곽 경계를 안 만든 채 중앙부만 먼저 과하게 꾸미기

---

## 6. 실제 작업 체크리스트
- 마을 기본 바닥이 전체에 깔렸는가
- 플레이어 스폰에서 출구까지 길이 보이는가
- 중앙 광장이 존재하는가
- 세이브 구역 바닥이 분리되어 있는가
- 상점가 세 NPC가 역할별로 구분되는가
- 포털 주변이 출구처럼 보이는가
- 외곽 경계가 시각적으로 설명되는가
- 장식이 동선을 가리지 않는가

---

## 7. 완료 기준
- `town.tscn`이 더 이상 테스트맵처럼 보이지 않는다
- 허브 역할이 한눈에 보인다
- 마을 안에서 `세이브`, `상점`, `출구`가 즉시 읽힌다
- 다음 단계로 `field_1` 작업을 이어갈 수 있다

---

## 8. 한 줄 결론
`town.tscn`은 `바닥 -> 길 -> 광장 -> 세이브 구역 -> 상점가 -> 출구 -> 경계 -> 소품` 순으로 작업해야 가장 적은 수정으로 가장 큰 효과를 얻는다.
