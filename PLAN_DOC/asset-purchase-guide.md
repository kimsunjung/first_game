# first_game 에셋 구매 가이드

## 문서 목적
- 현재 프로젝트에 맞는 에셋 구매 우선순위를 정한다.
- 충동 구매를 막고, 실제 부족한 부분만 보강하도록 한다.
- `Godot -> Unity 전환` 없이도 품질을 올릴 수 있는 구매 전략을 만든다.

작성일: 2026-04-15

---

## 1. 전제
현재 프로젝트는 이미 아래 리소스를 갖고 있다.
- 기본 월드 타일셋
- 캐릭터/몬스터 스프라이트 일부
- 오디오 기본 리소스
- UI 씬 구조
- 맵, 오브젝트, 상점, 세이브 포인트, 스킬 UI 등 게임 기능 구조

즉, 지금은 `무에서 유를 만드는 구매`가 아니라 `부족한 조각을 채우는 구매`가 맞다.

---

## 2. 결론 먼저
### 지금 가장 먼저 사야 할 가능성이 높은 것
1. 환경/타일셋 확장팩
2. UI 아이콘 또는 RPG UI 팩
3. 몬스터/NPC 확장팩

### 지금 당장 사지 말아야 할 것
- 올인원 RPG 템플릿
- 시스템 포함 대형 템플릿 팩
- 스타일이 다른 캐릭터 팩 여러 개
- 월드 기준이 정해지기 전의 무계획한 번들 구매

---

## 3. 구매 우선순위 상세
### 1순위: 환경/타일셋 확장팩
가장 추천한다.

이유:
- 탑다운 게임은 맵 품질이 첫인상을 크게 좌우한다.
- 현재 프로젝트는 이미 맵 구조가 있으므로, 환경 퀄리티 보강 효과가 즉시 나온다.
- 배경이 안정되면 캐릭터와 UI 판단도 쉬워진다.

사야 하는 경우:
- 현재 맵이 반복적으로 보여 단조롭다.
- `town`, `field`, `dungeon`이 같은 느낌으로 보여 구분이 약하다.
- 길, 바닥, 벽, 구조물 조합이 부족하다.

좋은 팩 조건:
- `16x16` 기준
- 탑다운 RPG 전용
- 같은 작가 또는 같은 시리즈 확장팩
- 바닥, 벽, 장식, 구조물이 한 계열로 포함됨

주의:
- 예쁜 단일 배경보다 실제 타일 조합 가능한 팩이 더 중요하다.

---

### 2순위: UI 아이콘 또는 RPG UI 팩
디자인 막막함을 줄이는 데 효과가 크다.

이유:
- UI는 코드보다 시각 완성도가 부족해 보이기 쉬운 영역이다.
- 인벤토리, 상점, 스킬 UI는 아이콘과 패널만 정리돼도 완성도가 급상승한다.
- 모바일 지향 프로젝트라 버튼 가독성이 중요하다.

사야 하는 경우:
- 장비, 소비 아이템, 스킬 아이콘이 부족하다.
- HUD, 인벤토리, 상점 패널이 지나치게 기본형으로 보인다.
- 버튼/패널 일관성이 부족하다.

좋은 팩 조건:
- 픽셀아트 기반 또는 현재 월드와 어울리는 단순한 RPG UI
- 아이콘 세트와 패널 세트가 함께 있거나 톤이 맞음
- 모바일에서도 읽기 쉬운 단순한 모양

주의:
- 지나치게 화려한 판타지 UI는 월드와 충돌할 수 있다.
- 현대적 플랫 UI를 섞으면 게임 톤이 깨질 수 있다.

---

### 3순위: 몬스터/NPC 확장팩
콘텐츠 다양성용이다.

이유:
- 현재 적과 NPC 종류가 더 늘어나면 RPG 느낌이 강해진다.
- 하지만 월드와 UI가 정리되기 전에는 체감 효율이 환경팩보다 낮다.

사야 하는 경우:
- 현재 적 조합이 너무 빨리 반복된다.
- 마을 NPC 역할 구분이 약하다.
- 던전/필드별 분위기 차이를 캐릭터로 주고 싶다.

좋은 팩 조건:
- 기존 `Pixel Crawler` 계열과 비율이 크게 다르지 않음
- Idle, Run, Attack, Death 등 필수 애니메이션 포함
- 몬스터 종류 간 실루엣 차이가 분명함

주의:
- 캐릭터 팩만 많이 사면 배경과 어울리지 않아 오히려 더 어색할 수 있다.

---

### 4순위: 이펙트 팩
좋지만 후순위다.

이유:
- 전투 타격감과 완성도를 높여준다.
- 그러나 현재 병목은 맵/스타일/아이콘 정리 쪽이라 우선순위는 낮다.

사야 하는 경우:
- 전투가 이미 재미있고, 이제 손맛을 보강하고 싶다.
- 스킬 사용감이 밋밋하다.

좋은 팩 조건:
- 픽셀아트 스타일 유지
- 과도한 파티클이 아닌 간결한 피격/폭발/힐 이펙트

---

### 5순위: 사운드/음원 팩
필수는 아니지만 상황에 따라 유효하다.

이유:
- 짧은 개발 시간 대비 체감 향상이 클 수 있다.
- 다만 비주얼 방향이 안 잡힌 상태에선 우선순위가 아니다.

---

## 4. 지금 사지 말아야 할 에셋
### 1. 올인원 RPG 템플릿
비추천.

이유:
- 이미 코드와 씬 구조가 많이 진행되어 있다.
- 템플릿을 사면 기존 구조와 충돌하거나 재구현 비용이 생길 가능성이 높다.
- 시스템을 돈으로 사는 순간 현재 구조를 버릴 유혹이 커진다.

### 2. 스타일이 다른 캐릭터 팩 여러 개
비추천.

이유:
- 월드 기준이 안 잡힌 상태에서 캐릭터를 늘리면 일관성이 무너진다.
- 특히 외곽선 두께와 채도가 다르면 같은 게임처럼 안 보인다.

### 3. 배경과 맞지 않는 고해상도 UI 팩
비추천.

이유:
- 픽셀 월드와 부자연스럽게 섞인다.
- 모바일에서 가독성도 애매해질 수 있다.

### 4. Unity 전용 툴 의존형 에셋
현재 비추천.

이유:
- 지금 프로젝트는 Godot 기준으로 충분히 진행돼 있다.
- 에셋 때문에 엔진을 바꾸는 건 손해일 가능성이 크다.

---

## 5. 구매 전 체크리스트
에셋을 사기 전에 아래 질문에 답한다.

### 환경팩 구매 전
- `town`과 `field_1`을 지금 리소스로 완성해봤는가
- 부족한 것이 바닥인지, 구조물인지, 소품인지 구분했는가
- 현재 타일셋과 색감이 맞는가

### UI 팩 구매 전
- HUD, 인벤토리, 상점 UI 중 어디가 가장 허전한가
- 아이콘 부족 문제인지 패널 스타일 문제인지 구분했는가
- 모바일 화면에서 읽을 수 있는 크기인가

### 캐릭터 팩 구매 전
- 정말 적 종류가 부족한가
- 아니면 색/스케일/맵 대비 문제인가
- 기존 팩과 자연스럽게 섞일 수 있는가

---

## 6. 추천 구매 순서
현재 프로젝트 기준 실제 추천 순서는 아래다.

1. `환경/타일셋 확장팩` 1개만 구매
2. 부족하면 `UI 아이콘 또는 UI 패널 팩` 1개 구매
3. 그다음 필요 시 `몬스터/NPC 팩` 구매
4. 마지막으로 `이펙트 팩` 검토

중요한 원칙:
- 한 번에 여러 팩을 사지 않는다.
- 하나 적용해보고 화면이 실제로 좋아졌는지 확인한 뒤 다음 구매를 결정한다.

---

## 7. 무료로 먼저 대체 가능한 것
지금은 구매 전에 아래를 먼저 활용한다.
- 현재 보유한 `Pixel Crawler` 리소스 정리
- 기존 `Kenney` 리소스를 최종용이 아닌 프로토타입용으로 제한 사용
- Godot 기본 UI에 색/패딩/폰트 규칙만 적용
- 맵 1개를 완성해 부족한 리소스를 구체적으로 목록화

---

## 8. 실제 의사결정 기준
### 지금 돈을 써도 되는 상태
- 맵 1개가 완성되어 있고
- 어디가 부족한지 명확히 말할 수 있으며
- 사려는 팩이 현재 스타일과 맞는지 판단 가능한 상태

### 아직 사면 안 되는 상태
- 그냥 불안해서 뭔가 사고 싶을 때
- 전체가 허전하다는 느낌만 있고, 부족한 요소가 특정되지 않았을 때
- 엔진 전환 욕구와 에셋 구매 욕구가 섞여 있을 때

---

## 9. 한 줄 결론
현재 `first_game`은 `환경 타일셋 -> UI 팩 -> 캐릭터/NPC 팩` 순으로 좁게 구매하는 전략이 가장 안전하다. 지금 단계에서 필요한 것은 대형 템플릿이 아니라 `스타일이 맞는 작은 보강`이다.
