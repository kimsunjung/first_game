# first_game 아트 방향서

## 문서 목적
- 현재 `Godot 4.6 + C# + 2D 탑다운 + 모바일` 구조에 맞는 비주얼 방향을 고정한다.
- 엔진 변경 없이도 디자인 의사결정을 빠르게 내릴 수 있도록 기준을 만든다.
- 에셋 구매 전에 무엇을 유지하고 무엇을 버릴지 정한다.

작성일: 2026-04-15

---

## 1. 프로젝트 현재 상태 요약
현재 프로젝트는 이미 다음 구조를 갖추고 있다.
- `Scenes/Maps`: `town`, `field_1`, `field_2`, `dungeon_1`, `dungeon_2`
- `Scenes/UI`: `hud`, `inventory_ui`, `character_window`, `shop_ui`, `mobile_controls`
- `Scripts`: `Core`, `Data`, `Entities`, `UI`로 역할이 나뉘어 있음
- `Resources`: `Tilesets`, `Kenney`, `Enemies`, `Items`, `Audio` 등 기본 리소스가 존재함

즉, 현재 병목은 엔진 부재가 아니라 아래 항목에 가깝다.
- 월드 스타일이 하나로 잠기지 않음
- 타일셋, 캐릭터, UI의 시각적 일관성이 약함
- 한 화면을 완성하는 기준보다 전체를 동시에 손대려는 위험이 있음

---

## 2. 비주얼 콘셉트
### 핵심 방향
- 장르 톤: 밝은 판타지 계열의 가벼운 액션 RPG
- 목표 인상: 복잡함보다 가독성 우선
- 우선 가치: 플레이어, 적, 상호작용 오브젝트가 즉시 구분되어야 함
- 플랫폼 고려: 모바일 화면에서 정보가 바로 읽혀야 함

### 피해야 할 방향
- 지나치게 어두운 다크 판타지
- 리얼풍 또는 반실사풍
- 서로 다른 팩을 무분별하게 섞은 콜라주 스타일
- 타일, 캐릭터, UI가 각각 다른 게임처럼 보이는 구성

---

## 3. 스타일 기준 축
### 메인 기준 에셋
`Pixel Crawler - Free Pack`을 메인 스타일 축으로 고정한다.

이유:
- 현재 리포지토리에 이미 많이 반영되어 있음
- 탑다운 RPG 문법과 잘 맞음
- 맵, 캐릭터, 오브젝트를 같은 계열로 맞추기 쉬움

### 보조 에셋 사용 규칙
`Kenney` 계열은 아래로 제한한다.
- 프로토타입용 임시 리소스
- 아이콘 또는 디버그용 대체 이미지
- 최종 월드 비주얼에는 원칙적으로 미사용

이유:
- `Kenney`와 `Pixel Crawler`는 외곽선, 면 분할, 볼륨감이 달라서 같이 쓰면 어색해질 가능성이 높음

---

## 4. 화면 스케일 규칙
- 기본 타일 크기: `16x16`
- 카메라 기준: `zoom 3x`
- 화면 우선순위: 배경보다 오브젝트와 캐릭터가 먼저 읽혀야 함

### 스케일 판단 기준
- 플레이어가 맵에 묻히면 실패
- 적이 배경과 구분되지 않으면 실패
- 버튼/패널 텍스트가 모바일에서 애매하면 실패

### 실무 원칙
- 스프라이트 문제는 우선 `padding`, `scale`, `camera zoom`으로 해결한다.
- 엔진 변경으로 해결할 문제로 확대 해석하지 않는다.

---

## 5. 색감 규칙
### 기본 팔레트 방향
- 바닥: 저채도 녹색, 갈색, 회갈색 위주
- 길: 바닥보다 약간 밝거나 따뜻한 톤
- 구조물: 바닥과 분리되도록 명도 차이를 둠
- 상호작용 오브젝트: 배경보다 더 눈에 띄는 색 사용

### 포인트 컬러
포인트 컬러는 하나만 메인으로 잡는다.
- 후보 1: 금색 계열
- 후보 2: 청록 계열

둘 중 하나만 선택해서 아래에 공통 적용한다.
- 선택 버튼
- 강조 UI
- 인터랙션 가능 오브젝트
- 퀘스트 또는 핵심 알림 요소

### 금지 규칙
- 강한 색을 여러 군데 동시에 사용하지 않는다.
- 배경이 캐릭터보다 더 눈에 띄게 만들지 않는다.

---

## 6. 맵 디자인 원칙
### 첫 완성 목표
전체 월드를 동시에 손대지 말고 아래 중 하나만 완성한다.
- `town`
- `field_1`

### 맵 구성 우선순위
1. 이동 경로가 보이는가
2. 전투/안전 구역이 구분되는가
3. 상호작용 지점이 잘 보이는가
4. 장식이 과하지 않은가

### 타일 사용 규칙
- 한 화면에서 핵심 바닥 타일은 소수만 사용
- 장식용 소품은 적게 넣고 큰 형태부터 잡음
- 맵이 허전해 보여도 먼저 동선과 읽기 쉬움을 맞춘다

---

## 7. 캐릭터와 적 디자인 원칙
### 플레이어
- 항상 가장 먼저 눈에 띄어야 함
- 배경보다 높은 명도 또는 더 분명한 실루엣 유지
- 공격 모션은 화려함보다 방향성과 판정감이 중요

### 적
- 적 종류별 실루엣 차이가 분명해야 함
- 작은 색 차이보다 전체 형태 차이를 우선함
- 적이 배경색과 섞이면 색 보정 또는 외곽 대비를 조정함

### NPC
- 적과 달리 위협적이지 않은 색과 자세 사용
- 상점 NPC, 스킬 NPC, 대장장이 NPC는 역할이 즉시 드러나야 함

---

## 8. UI 방향
### 목표
- 예쁜 UI보다 먼저 읽히는 UI
- 패널, 버튼, 강조색 규칙이 통일된 UI

### 적용 규칙
- 패널 배경: 어두운 반투명 계열
- 버튼: 공통 높이, 공통 모서리, 공통 강조색 사용
- 텍스트: 최대한 짧고 크게
- HUD: 체력, 마나, 골드, 퀵슬롯 정보만 우선 명확히 표현

### UI 스타일 금지 항목
- 패널마다 다른 색 체계 사용
- 장식이 많은데 정보는 덜 읽히는 구성
- 월드와 맞지 않는 과도한 현대적 미니멀 UI

---

## 9. 에셋 혼합 규칙
다른 팩을 추가할 때는 아래 세 가지를 동시에 확인한다.
- 비율이 맞는가
- 채도와 명도가 맞는가
- 외곽선 두께나 디테일 밀도가 맞는가

하나라도 어긋나면 바로 메인 월드에 투입하지 않는다.

### 안전한 추가 순서
1. 환경 타일
2. UI 아이콘
3. 몬스터 또는 NPC
4. 이펙트
