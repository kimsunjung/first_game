# first_game 맵/아트 작업 우선순위 계획

## 문서 목적
- 현재 `town.tscn`과 `field_1.tscn`의 상태를 기준으로 실제 작업 순서를 정한다.
- 어디부터 꾸며야 체감 효과가 큰지 결정한다.
- 맵, UI, 에셋 구매를 한 흐름으로 연결한다.

작성일: 2026-04-15

---

## 1. 현재 상태 요약
현재 `Scenes/Maps/town.tscn`과 `Scenes/Maps/field_1.tscn`은 구조적으로는 플레이 가능하지만, 비주얼은 아직 플레이스홀더 성격이 강하다.

### `town.tscn` 상태
- 배경이 `ColorRect` 기반의 단색 처리
- `ShopNPC`, `SkillShopNPC`, `BlacksmithNPC`, `SavePoint`, `PortalToField1`가 배치되어 있음
- 즉, `마을 허브` 역할은 이미 준비되어 있음

### `field_1.tscn` 상태
- 배경이 `ColorRect` 기반의 단색 처리
- `SavePoint`, `PortalToTown`, `PortalToDungeon1`, `PortalToField2`, `EnemySpawner`가 배치되어 있음
- 즉, `야외 전투/이동 필드` 역할은 이미 준비되어 있음

이 상태는 기능 확인에는 충분하지만, 플레이 경험상 공간성이 부족하고 게임 세계가 비어 보일 수 있다.

---

## 2. 우선순위 결론
### 1순위: `town`
가장 먼저 손대야 한다.

이유:
- 플레이어가 자주 돌아오는 허브 공간이다.
- 상점, 강화, 스킬, 세이브 같은 핵심 루프가 여기에 모여 있다.
- `town`이 완성되면 게임의 전체 인상이 바로 좋아진다.

### 2순위: `field_1`
`town` 다음으로 작업한다.

이유:
- 마을과 던전을 연결하는 첫 전투 필드다.
- 플레이어가 게임의 바깥 세계를 처음 체감하는 구역이다.
- 동선과 전투 구역의 읽기 쉬움이 중요하다.

### 3순위: UI 톤 정리
맵 1개가 어느 정도 갖춰진 뒤 병행한다.

이유:
- 월드가 정리된 다음 UI를 맞추면 전체 인상이 맞아떨어진다.
- 반대로 UI를 먼저 지나치게 고치면 월드가 계속 임시처럼 느껴질 수 있다.

---

## 3. `town` 작업 계획
### 목표
`town`을 `기능 허브`가 아니라 `실제 마을처럼 느껴지는 공간`으로 만든다.

### 핵심 구성 목표
- 플레이어 입장 지점이 분명해야 함
- 상점 구역과 세이브 구역이 시각적으로 구분되어야 함
- 포털이 배경에 묻히지 않아야 함
- 이동 경로가 자연스러워야 함

### 추천 작업 순서
#### 1단계: 바닥과 길
- 단색 배경을 타일 기반 바닥으로 교체
- 포털에서 마을 중심으로 이어지는 메인 길 생성
- 상점 NPC 3명 앞쪽에 작은 광장 또는 상점 거리 느낌 추가

#### 2단계: 구역 구분
- `SavePoint` 주변은 조용한 성역 느낌으로 분리
- `ShopNPC`, `SkillShopNPC`, `BlacksmithNPC`는 같은 줄에 있어도 배경 소품으로 역할 차이를 준다
- 예시:
  - 상점: 천막, 상자, 표지판
  - 스킬 상점: 책, 룬, 배너
  - 대장간: 모루, 화로, 금속 소품

#### 3단계: 경계 처리
- 맵 외곽을 단순 충돌벽 대신 울타리, 건물, 나무, 절벽 등으로 시각화
- 플레이어가 `왜 더 못 가는지` 이해할 수 있어야 함

#### 4단계: 마을 포인트 만들기
- 중앙 또는 한쪽에 시선이 모이는 구조물 1개 배치
- 예시:
  - 우물
  - 큰 나무
  - 조형물
  - 작은 광장

#### 5단계: 소품 최소 보강
- 상자, 배럴, 표지판, 화분, 램프 정도만 사용
- 처음부터 장식 과밀도를 높이지 않는다

### `town` 완료 기준
- 상점 구역, 세이브 구역, 출구 구역이 한눈에 구분됨
- NPC 세 명이 그냥 서 있는 게 아니라 각자 역할 공간을 가짐
- 포털과 세이브 포인트가 눈에 잘 띔
- 단색 테스트맵 느낌이 사라짐

## 4. `field_1` 작업 계획
### 목표
`field_1`을 `전투와 이동이 동시에 가능한 첫 야외 필드`로 만든다.

### 핵심 구성 목표
- 마을 입구에서 필드 중심, 던전 입구, 다음 필드로 가는 흐름이 보여야 함
- 전투 공간이 플레이어 이동 동선과 충돌하지 않아야 함
- 저장 지점이 맵 맥락상 자연스러워야 함

### 추천 작업 순서
#### 1단계: 세 개의 주요 동선 연결
- `PortalToTown`에서 출발하는 길
- `PortalToDungeon1`으로 이어지는 길
- `PortalToField2`로 내려가는 길
- 이 세 경로는 시각적으로 연결돼야 함

#### 2단계: 안전 구역과 전투 구역 분리
- 마을 포털 근처는 비교적 안전한 구역으로 둔다
- `EnemySpawner` 중심부는 전투 구역 느낌으로 만든다
- 바닥 색, 풀 밀도, 장애물 밀도로 구역 차이를 준다

#### 3단계: 랜드마크 추가
- 저장 포인트 주변에 작은 구조물이나 제단 느낌 추가
- 던전 포털 주변은 더 위험하고 어두운 분위기 부여
- 다음 필드로 내려가는 쪽은 길이 이어진다는 인상을 줌

#### 4단계: 장애물과 동선 정리
- 나무, 바위, 울타리 등으로 필드를 전부 막지 말고 흐름만 유도
- 전투 중 회피 공간이 남아 있어야 함
- `EnemySpawner` 근처는 지나치게 복잡하게 만들지 않는다

#### 5단계: 장식 최소화
- 꽃, 풀, 작은 돌, 나무 그루터기 정도로 끝낸다
- 핵심은 장식이 아니라 `필드 구조`다

### `field_1` 완료 기준
- 세 방향 포털의 이동 목적이 분명함
- 적이 등장하는 구역과 지나가는 길이 헷갈리지 않음
- 저장 포인트가 뜬금없이 놓인 오브젝트처럼 보이지 않음
- 첫 필드로서 `게임이 시작됐다`는 느낌을 줌

---

## 5. 맵 작업 중 하지 말아야 할 것
- `town`, `field_1`, `dungeon_1`을 동시에 꾸미기 시작하는 것
- 장식물만 많이 놓고 동선은 나중에 맞추는 것
- 타일셋 여러 종류를 한 맵에 과도하게 섞는 것
- 전투 공간에 시각 노이즈를 너무 많이 넣는 것

---

## 6. UI 작업 병행 시점
UI는 `town`의 1차 완성 후 시작하는 것을 추천한다.

### 먼저 손댈 UI
1. `HUD`
2. `inventory_ui`
3. `shop_ui`
4. `skill_shop_ui`

### 이유
- `town`은 상점과 허브 UI를 가장 많이 확인하는 공간이다.
- 마을 화면이 어느 정도 정리된 후 UI를 붙이면 전체 톤이 더 잘 맞는다.

---

## 7. 에셋 구매와 연결되는 판단 기준
### `town` 작업 후 구매 검토
아래가 느껴지면 환경팩 검토 가치가 있다.
- 건물/가게/소품 종류가 부족함
- 마을을 구분할 장식 리소스가 적음

### `field_1` 작업 후 구매 검토
아래가 느껴지면 환경 확장이나 적/NPC 팩을 검토할 수 있다.
- 필드 타일이 반복적임
- 적 종류가 너무 빨리 반복됨
- 랜드마크가 부족해 공간 기억점이 약함

### 그 전에는
먼저 무료 리소스와 현재 보유 리소스로 최대한 완성해본다.

---

## 8. 실제 실행 순서
1. `town` 바닥/길/구역 분리 작업
2. `town` 경계와 소품 최소 정리
3. `town` 기준 UI 톤 확인
4. `field_1` 동선과 전투 구역 정리
5. `field_1` 랜드마크와 최소 소품 추가
6. 부족한 리소스를 목록화
7. 그 뒤에만 구매 결정

---

## 9. 최종 한 줄 결론
현재 `first_game`의 아트 작업은 `town 먼저`, `field_1 다음`, `UI는 그 뒤 병행`이 가장 효율적이다. 지금은 넓게 퍼지는 것보다 `허브 1개 + 필드 1개 완성`이 맞다.
